██ SECTION 3 — BACKEND & API RULES
3.1 — STRICT 502 GATEWAY FALLBACK (ANTI-SHATTER PROTOCOL)
All proxy routing functions forwarding traffic to backend microservices MUST
enclose async HTTP client requests (httpx.AsyncClient) in comprehensive try/except
blocks. On connection failure (timeout, refused, offline):

FORBIDDEN: allowing ASGI thread to shatter into raw 500
FORBIDDEN: swallowing the error with fail-open 200 OK
REQUIRED: explicit trap returning JSONResponse(status_code=502,
content={"error": "Gateway Unreachable", "detail": str(exception)})

3.2 — UNIFIED I/O SERIALIZATION ENVELOPE
All paginated collection routes MUST return this exact JSON envelope:
{"items": [...], "total": <int>, "limit": <int>, "offset": <int>}
Fragmented structures (arbitrary data keys, redundant "status": "success" strings)
are permanently forbidden.
3.3 — STRICT PAGINATION (NO UNBOUNDED QUERIES)
SELECT * FROM table is permanently forbidden. Every collection GET route natively
ingests and enforces limit and offset query parameters bound directly to the DB
execution layer. Frontend Axios must pass these parameters. Masking DOM bloat with
CSS overflow:hidden is a fatal architecture violation.
3.4 — EXPLICIT COLUMN ENUMERATION
Every SQL query must explicitly enumerate exact columns required. Fetching * or
unused columns to prevent memory inflation and DB-to-API serialization coupling
is forbidden.
3.5 — BACKEND ROUTING (NO CATCH-ALL, NO STATIC SERVING)
FastAPI routers are strict JSON I/O engines. Forbidden:

Catch-all fallback routes (/{full_path:path})
StaticFiles mounting to serve frontend assets
All routing must be explicitly defined. Unregistered paths fall through to strict
404 Not Found JSON exceptions.

3.6 — SAST BEFORE DISK I/O
All pre-flight static analysis, doctrine checks, and security validations must be
bound to the backend mutation endpoint (e.g., /api/v1/atomizer/mutate) prior to
disk I/O. Client-side React validation is strictly a visual aid — never the
primary security boundary. All pipelines must natively handle HTTP 403 exceptions.
3.7 — OBJECT-LEVEL AUTHORIZATION
All state-mutating DB operations must execute instance-level ownership checks
against the RS256 JWT payload (e.g., asserting assigned_tech == jwt_payload['sub'])
within the transaction block before mutation. RBAC alone without discrete
object-level ownership checks is a fatal authorization bypass.
3.8 — NGINX TRAILING SLASH DOCTRINE
All Nginx blocks proxying to internal microservices MUST enforce trailing slashes
on both location boundary and proxy_pass directive:
location /auth/ { proxy_pass http://127.0.0.1:9000/; }
Omitting trailing slash causes silent URI stripping and fatal backend routing errors.

Enterprise data tables exceeding 5 columns MUST implement a data-label CSS reflow

touch-friendly Key-Value cards on viewports < 900px.
4.7 — CATEGORICAL DATA — NO FREE-TEXT DOM INPUTS
All bounded enterprise domains (Categories, Departments, Status Enums) MUST use

DOM inputs for categorical data are forbidden. Backend engine synthesizes all PKs;
frontend payload must never include a PK on ingestion actuations.
