Chapter 06

Build & Deploy Pipeline

How the factory creates new applications from natural language — from prompt to production.

🚀 App Generation Flow

  1. Prompt Intake — Describe the app in natural language via the Factory UI or API (POST /api/build)
  2. Blueprint Selection — Factory selects from templates: gemini_reasoner, multi_agent_core, elite_council, file_factory, sentinel_bridge
  3. Code Generation — app_generator.py scaffolds the full stack: FastAPI server, React frontend, n8n webhook, requirements.txt
  4. V3 Injection — v3_retrofit.py injects the resilience header, preflight checks, and safe_post into every Python file
  5. Auto-Heal Layer — v3_autoheal_inject.py upgrades safe_post() to healed_post() with retry + diagnosis
  6. Docker Packaging — Auto-generates Dockerfile, docker-compose.yml, and .dockerignore
  7. Validation — v3_safety_check.py compile-checks all generated files — rollback if any fail
  8. Registration — App added to registry.json with port, webhook, and n8n workflow ID
  9. Launch — Batch file auto-generated, desktop shortcut created, server started

⚠️ Credential Lifecycle
The V3 Auto-Heal engine includes credential decay detection. If the n8n API key expires or is rotated, the diagnose() function detects the 401 response and flags CREDENTIAL_DECAY — preventing silent failures across all child apps.
