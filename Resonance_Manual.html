01 — The Alex Persona & Council Architecture

Resonance is a multi-agent educational companion built for a 16-year-old male with auditory processing challenges and speech delays. At its core, it is not a single AI — it is a Council of Therapists working behind a unified persona.

🧠 Alex — The Primary Persona

Alex is the user-facing identity. All interactions appear to come from Alex — warm, patient, and engaging. The persona is designed to feel like a cool older friend, not a clinical assistant.

  • Voice — Casual, encouraging, uses contractions ("that's awesome!" not "that is awesome")
  • Sentences — Short (10-15 words), clear, no jargon. Never double-negatives.
  • Engagement — Celebrates effort, not just correctness. Uses humor and real-world analogies.
  • Identity Lock — Always responds "I'm Alex!" to identity questions. No exceptions.
  • Teach-Back Loop — Asks "Can you tell me what that means in your words?" to reinforce learning.

👥 The Council of Therapists (Behind the Scenes)

Four specialized agents operate behind Alex's unified voice. The user never sees them directly — they influence Alex's behavior through system prompt injection.

🎭 Alex (The Communicator) PRIMARY

The face of the system. Handles direct conversation, personality, engagement patterns, kinetic typography, Wingman Mode, and the Mixing Board for language construction.

👁️ The Observer SENTIMENT

Monitors emotional state through sentiment analysis. Detects frustration patterns (consecutive short or all-caps messages), confidence levels, and engagement dips. Triggers the "Pivot Protocol" when frustration is detected — suggests a game, changes topic, or offers encouragement.

🏗️ The Scaffolder MODALITY

Adapts content delivery modality based on the user's learning profile. Provides visual aids, step-by-step breakdowns, analogies, and fill-in-the-blank exercises. Adjusts complexity in real-time based on response patterns.

🛡️ The Overseer ORCHESTRATOR

Coordinates the other three agents. Manages turn-taking, ensures Alex stays on-topic with parent directives, enforces safety boundaries, and handles the Sandbox Mode for parent testing.

03 — Parent Portal — Administrative Control

The Parent Portal gives guardians and educators full control over Alex's behavior without requiring technical knowledge. Access is protected by a 4-digit PIN and visually distinguished by an amber accent color scheme.

🔐 PIN-Based Access Control

The portal uses a simple 4-digit PIN overlay that prevents the student from modifying Alex's instructions:

  • PIN Gate — Full-screen modal with numeric input. Incorrect attempts show shake animation.
  • Default PIN — Set to 1234 on first install. Can be changed via the Settings tab.
  • PIN Reset — POST /api/parent/pin-reset with current_pin and new_pin.
  • No Timeout — Session stays unlocked until the parent explicitly closes the portal.

🧪 Sandbox Mode

Parents can activate Sandbox Mode to test Alex's behavior without affecting the student's progress data or profile hints. When active, a "SANDBOX" badge appears in the chat header.

💡 How Sandbox Works
The system prompt includes a --- SANDBOX MODE (ACTIVE) --- flag. Alex treats the conversation as a test session — no profile updates, no progress logging, no hint collection. The parent can verify that their instructions produce the desired behavior before going live.

📊 Automated Progress Tracking

After every streaming response, app_stream.py automatically logs:

  • Engagement Timestamps — When the session occurred and which channel was used.
  • Verbal Flux — Word counts for both user messages and Alex's responses.
  • Session Intensity — Classified as Short (<100 words), Medium (100-300), or Long (>300).
  • Retention — Last 100 entries are maintained in parent_config.json.

04 — Clinical Intelligence & Dr. Aris Integration

Resonance includes a clinical intelligence pipeline that processes professional reports (medical, educational, behavioral) and integrates findings into Alex's therapeutic context. This system is powered by Dr. Aris, the Specialist Agent for developmental science.

📄 Report Upload Pipeline

Parents upload professional reports through the Parent Portal's Clinical Intelligence tab:

Upload PDF/DOCX
→
Text Extraction
→
AI Digest Engine
→
Dr. Aris Analysis
→
Alex Directives
  1. Upload — POST /api/parent/report-upload accepts PDF, DOCX, TXT files with a report type tag (medical, educational, behavioral, therapy).
  2. AI Digest — Gemini extracts diagnoses, accommodations, triggers, strengths, and recommended strategies from the report text.
  3. Dr. Aris Review — GET /api/parent/dr-aris/analysis returns structured analysis with developmental insights.
  4. Proposal System — Dr. Aris generates actionable proposals (e.g., "Increase visual scaffold for math concepts"). Parents approve or reject each via POST /api/parent/dr-aris/proposals/{id}/approve.
  5. Directive Injection — Approved proposals become directives injected into Alex's system prompt via the dr_aris_directives array.

🏥 Dr. Aris Boardroom

The Dr. Aris Boardroom provides a direct query interface for parents to ask developmental questions:

  • Boardroom Query — POST /api/dr-aris/boardroom-query — Ask any developmental or clinical question about the student.
  • History — GET /api/dr-aris/boardroom-history — View past consultations.
  • Full Report — GET /api/parent/dr-aris/report — Comprehensive developmental report aggregating all uploaded reports and analysis.
  • Alerts — GET /api/parent/dr-aris/alerts — Flags when report content reveals urgent concerns.
⚠️ Privacy by Design
All clinical data stays in the local parent_config.json and dr_aris_reports.json. No clinical information is sent to external databases. Gemini API calls use the raw text only for analysis — nothing is stored in Gemini's context beyond the session.

08 — Deployment, Launch & Troubleshooting

🚀 Launch Protocol

Resonance is a dual-service application (FastAPI backend + React/Vite frontend). Launch via the batch file:

  1. Double-click Launch_Resonance_V3.bat (or the Desktop shortcut).
  2. The script detects the Google Drive path, installs dependencies (pip install -r requirements.txt, npm install).
  3. Activates the n8n workflow (IT4LuCQm7Ph62Hf2) via n8n_lifecycle.py.
  4. Starts server.py on Port 5006.
  5. Starts the Vite dev server on Port 5174+.
  6. Opens the browser at http://localhost:5174.
