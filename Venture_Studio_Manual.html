Step-by-step procedures for the three core operational workflows: ingesting legacy materials, using the Boardroom persuasion loop, and driving the design engine.

📥 The Legacy Ingest

How to introduce existing business materials into the Antigravity ecosystem for AI-powered analysis and transformation:

  1. Prepare Materials — Gather all documents (pitch decks, financials, research PDFs). Rename them with a clear convention: PROJECT_type_version.ext
  2. Upload via Builder Chat — Open the Meta App Factory UI → Builder Chat → click 📎 Upload. The file is sent to the DocumentParserService for AI-powered text extraction (PDF, DOCX, TXT, CSV) and automatic categorization (Legal, Finance, Ops, Medical, Technical). A parse result toast appears showing category, confidence score, and routing destination. The summary is injected into the chat context for immediate analysis.
  3. Upload via Sentinel Bridge — Sentinel's dashboard (Actions → 📄 Upload & Parse Document) extracts every individual task, obligation, and deadline from the document using Gemini AI, and creates a separate reminder for each — with its own description, due date, category, and priority. High-priority items trigger push notifications.
  4. Trigger the Critic's Audit — Prompt: "Analyze the uploaded document. Run a full CRITIC audit — stress-test the assumptions, identify gaps, and score confidence 1-10." The system routes to the CRITIC agent via the Elite Council.
  5. Review the Draft Summary — The Council returns a draft_summary with actionable insights. You can approve to proceed or pivot with new instructions.
  6. Store to Vector Memory — Key findings are automatically stored via the vector_memory tool (ChromaDB). Future prompts can reference: "Recall the CRITIC's analysis from last week."
⚠️ File Size Limit
The upload truncates at 3000 characters. For larger documents, break them into sections and upload sequentially, or use Google Workspace integration to ingest directly from Drive.

🎭 The Persuasion Loop (War Room UI)

The Persuasion Loop now operates through the Adversarial War Room — a real-time WebSocket interface at localhost:5173 → War Room. See Chapter 5 for the full War Room guide and Chapter 6 for the Socratic Challenger Protocol.

  1. Set a Debate Topic — Open the War Room sidebar (⚔️ icon). In the topic bar, enter your directive and click 🏛️ Debate to open a free boardroom session.
  2. Trigger a Socratic Challenge — Enter a score (1–10) and click 🔍 Challenge to invoke a Strategic Pause. If the score is below 9.5, the Critic presents 3 state-of-the-art weaknesses.
  3. Watch the Board Deliberate — CEO, CMO, CFO, and CRITIC respond in real-time in the Dialogue Feed. Each agent is color-coded with their persona icon.
  4. Present Evidence — In Convince Mode, the intervention box switches to an evidence input. Use data-driven language: metrics, validated research, A/B test results. The Critic scores your reasoning out of 10.
  5. Hard Override — If the Board resists, click 🚨 Force Proceed. Risks are logged with severity (LOW → CRITICAL) and a 30-day audit deadline is set for HIGH/CRITICAL overrides.
🚨 Hard Override Warning
Force Proceed bypasses the Critic's quality gates and logs the override in socratic_audit.json. HIGH and CRITICAL overrides trigger a mandatory 30-day audit review.

Boardroom Integration & Automation (Phase 4)

Automated file ingestion, legacy workflow cleanup, and system-wide audits — all wired into the War Room for real-time oversight.

📥 Incoming File Watcher

incoming_watcher.py monitors projects/*/incoming/ directories on a 60-second polling interval. When new files are detected:

  1. File Detected — Hash-based deduplication prevents re-processing. Each file is fingerprinted by path + mtime + size.
  2. War Room Seeded — A boardroom session is automatically opened via POST /api/warroom/seed with the file details as the topic.
  3. Socratic Challenge Issued — A challenge is triggered via POST /api/warroom/challenge with a starting score of 5.0 (skeptical). The Critic presents 3 weaknesses in the War Room feed.
  4. Commander Review — You see the challenge in the War Room and respond with evidence or Force Proceed.
📁 Auto-Start
The watcher starts automatically when the FastAPI server boots. It runs as a daemon thread with a 60-second poll interval. To run a one-shot scan: python incoming_watcher.py --once

🗄️ n8n Legacy Archiver

n8n_archiver.py uses the Antigravity_Full_v2 API key to scan all n8n Cloud workflows and identify legacy entries matching V1, TEST, or OLD labels.

📋 MASTER_INDEX Socratic Audit

The POST /api/audit/master_index endpoint performs a full Socratic Audit on MASTER_INDEX.md:

  1. Parse — Reads the full MASTER_INDEX.md and counts sections, healing cycles, deployments, errors, and fresh boots.

  3. Challenge — Triggers a Socratic Challenge at score 6.0. The Critic presents 3 structural weaknesses based on the index analysis.
  4. Await Input — The board enters STRATEGIC PAUSE at 6/10 (Gap: 3.5). The Commander must present evidence or Force Proceed.
